Ответы на вопросы для самоконтроля - ПР24
1. Какие методы HTTP используются для CRUD операций и в каких случаях?
- **GET** - получение данных (чтение списка задач)
- **POST** - создание новых данных (регистрация, создание задачи)
- **PUT/PATCH** - обновление данных (изменение статуса задачи)
- **DELETE** - удаление данных (удаление задачи)
2. Как организовать обработку ошибок на фронтенде при работе с API?
- Использовать try/catch блоки в асинхронных функциях
- Отображать пользовательские сообщения об ошибках
- Использовать состояния для управления ошибками (useState)
- Логировать ошибки в консоль для отладки
3. Что такое CORS и как его настроить для разработки?
**CORS** (Cross-Origin Resource Sharing) - механизм безопасности браузера, который ограничивает кросс-доменные запросы.
**Настройка:**
```javascript
const cors = require('cors');
app.use(cors());
4.Какие подходы к управлению состоянием при интеграции с API вы знаете?
Локальное состояние (useState) - для простых случаев
Context API - для глобального состояния
Redux - для сложных приложений
React Query/SWR - для кэширования API запросов
5. Как реализовать индикацию загрузки без ухудшения UX?
Показывать спиннер/лоадер во время запросов
Использовать состояния loading (true/false)
Оптимистичные обновления - сразу показывать изменения
Скелетон-экраны вместо спиннеров
6. В чем преимущества и недостатки оптимистичных обновлений?
Преимущества:
Лучший UX (мгновенный отклик)
Уменьшение воспринимаемой задержки
Недостатки:
Нужно откатывать изменения при ошибках
Сложнее в реализации
7. Как организовать валидацию данных на клиенте и сервере?
Клиент:
HTML5 валидация (required, type="email")
JavaScript валидация перед отправкой
Валидация в реальном времени
Сервер:
Валидация в middleware
Проверка обязательных полей
Валидация форматов данных
8. Какие методы кэширования данных с сервера можно использовать?
LocalStorage - для токенов и пользовательских данных
React Query - автоматическое кэширование API запросов
Service Workers - для офлайн-работы
HTTP кэширование - заголовки Cache-Control
