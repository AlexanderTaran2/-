1. Viewport meta tag - HTML-тег для управления масштабированием на мобильных устройствах. Важен для корректного отображения медиа-запросов и предотвращения автоматического масштабирования.
2. Основные подходы: Fluid (резиновые проценты), Adaptive (фиксированные брейкпоинты), Responsive (комбинация обоих + Flexbox/Grid).
3. Mobile First - разработка сначала для мобильных, затем адаптация для десктопа через min-width медиа-запросы. Преимущества: лучшая производительность, прогрессивное улучшение, фокус на контенте.
4. Предпочтительные единицы: % для контейнеров, rem/em для типографики, vw/vh для viewport-зависимых элементов, clamp() для адаптивных шрифтов.
5. clamp() создает плавно изменяющиеся значения: clamp(мин, предпочтительный, макс). Пример: font-size: clamp(1rem, 2.5vw, 2rem) - шрифт от 1rem до 2rem.
6. Основные брейкпоинты: 1024px (планшеты), 768px (мобильные/планшеты), 480px (маленькие мобильные). Определены по статистике популярных устройств.
7. Медиа-запросы @media применяют CSS при определенных условиях. Пример: @media (max-width: 768px) { .menu { display: none; } }
8. max-width vs min-width: max-width - стили для экранов МЕНЬШЕ значения (Desktop First), min-width - для экранов БОЛЬШЕ значения (Mobile First).
9. Адаптивные изображения: max-width: 100%, height: auto, object-fit: cover. Дополнительно: srcset для разных разрешений, picture для art direction.
10. Инструменты тестирования: Режим адаптивного дизайна в DevTools, эмуляция устройств, throttle для скорости сети, Lighthouse для аудита, Console для отладки.
