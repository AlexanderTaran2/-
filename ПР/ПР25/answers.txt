Ответы на вопросы для самоконтроля - ПР25
1. Какие основные различия между development и production окружениями?
**Development:**
- Подробные логи ошибок
- Горячая перезагрузка (hot reload)
- Тестовые данные и моки
- Медленнее, но удобнее для разработки
**Production:**
- Минимизированный и оптимизированный код
- Сжатые ресурсы (CSS, JS, изображения)
- Безопасные настройки
- Настроенное кэширование
- Мониторинг и логирование
2. Почему важно использовать environment variables в production?
- **Безопасность**: Секреты (API ключи, JWT секреты) не попадают в код
- **Гибкость**: Разные настройки для разных сред без изменения кода
- **Соответствие стандартам**: 12-factor app principles
- **Упрощение деплоя**: Легко менять настройки между средами
3. Как организовать управление разными средами (dev/staging/prod)?
- **Файлы окружения**: `.env.development`, `.env.staging`, `.env.production`
- **Переменные платформ**: Environment variables в Vercel/Railway
- **Флаги функциональности**: Feature flags для контроля доступа
- **Разные базы данных**: Отдельные БД для каждой среды
4. Какие методы оптимизации производительности при деплое вы знаете?
- **Минификация**: Удаление пробелов, комментариев из кода
- **Сжатие**: Gzip, Brotli для ресурсов
- **Кэширование**: CDN, browser caching
- **Ленивая загрузка**: Code splitting в React
- **Оптимизация изображений**: WebP, сжатие
- **Bundle analysis**: Анализ размера бандлов
5. Как настроить мониторинг ошибок в production?
- **Sentry/LogRocket**: Для отслеживания ошибок фронтенда
- **Application Insights**: Мониторинг производительности
- **Custom logging**: Логирование в базу данных или файлы
- **Uptime monitoring**: Ping-сервисы для проверки доступности
- **Performance metrics**: Google Analytics, Lighthouse
6. Что такое CI/CD и какие преимущества это дает?
**CI (Continuous Integration)** - автоматическое тестирование каждого коммита
**CD (Continuous Deployment)** - автоматический деплой после успешных тестов
**Преимущества:**
- Быстрые и частые релизы
- Меньше ручных ошибок
- Автоматическое тестирование
- Быстрое обнаружение багов
7. Как организовать процесс обновления приложения без downtime?
- **Blue-green deployment**: Две идентичные среды, переключение трафика
- **Canary releases**: Постепенный rollout новым пользователям
- **Feature flags**: Включение функциональности без деплоя
- **Database migrations**: Обратимые миграции без блокировок
8. Какие меры безопасности важно соблюдать при деплое?
- **HTTPS**: Обязательное шифрование трафика
- **Environment variables**: Хранение секретов вне кода
- **Security headers**: CSP, HSTS, X-Frame-Options
- **Dependency scanning**: Проверка уязвимостей в пакетах
- **Access control**: Минимальные привилегии для сервисов
- **Regular updates**: Своевременное обновление зависимостей
